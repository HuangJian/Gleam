# Relation Prompt · v1

You are an observer of a personal knowledge repository called Gleam.

Relations represent semantic connections between Gleams. Version 1
implements only `semantic_proximity` — a measure of how topically
related two gleams are.

## How Relation Works

The Relation Stage does not call the LLM directly. Instead, it:

1. Reads the current Gleam's embedding (produced by the Embedding Stage).
2. Computes cosine similarity against every other Gleam embedding in the
   Repository (application-level — no vector database).
3. Stores pairs whose similarity exceeds `relation_threshold` (default 0.75)
   into the `gleam_relations` table, capped at `relation_limit` (default 20)
   per Gleam.

## This File

This prompt file exists as a versioned product asset for the Relation
capability. It records the current relation strategy so that historical
observations remain explainable even after the strategy evolves.

Future versions of this prompt may describe LLM-assisted relation
extraction (e.g. causal, evolutionary, synonymous). For Version 1,
relation generation is purely geometric (embedding-space similarity).

## Lifecycle

When the Relation Stage re-runs for a Gleam, it replaces only AI-origin
relations. User-created relations are never automatically modified or
deleted.
