## What I do

1. Engage the user in a thorough discussion to clarify requirements, constraints, edge cases, and preferences
2. Analyze technical options, tradeoffs, and propose a solution design before writing code
3. Output a `<task>.plan.md` file in the project root containing:
   - Goals and non-goals
   - Design decisions and rationale
   - Step-by-step implementation plan
   - Files to be created or modified
   - Open questions or risks
4. Present the plan to the user for review and approval before implementation

## When to use me

Use this skill when the user's request is non-trivial, ambiguous, or benefits from upfront design before coding. Do NOT use for trivial or well-understood tasks (typo fixes, simple CSS tweaks, straightforward refactors, etc.).
