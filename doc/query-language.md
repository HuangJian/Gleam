# 拾光查询语言（Gleam Query Language）

拾光的搜索框支持一套轻量级的查询语法，让你按关键字、标签、来源域名、来源类型、时间等维度精确筛选「微光」。这套语法是产品「Recall（回溯）」能力的一部分。

> 直接在回顾界面的搜索框中输入查询语句即可。留空时显示全部微光。

## 一、基本写法

查询由若干「子句」组成，子句之间默认是 **且（AND）** 关系。

| 写法                | 含义       | 匹配规则                                                               |
| ------------------- | ---------- | ---------------------------------------------------------------------- |
| `react`             | 裸关键字   | 在「理解内容、标签、来源标题、来源摘录」中做**不区分大小写**的子串匹配 |
| `text:统计`         | 正文关键字 | 仅在「理解内容 + 来源摘录」中做子串匹配                                |
| `title:深度`        | 来源标题   | 在来源标题中做子串匹配                                                 |
| `#react`            | 标签       | 与任一标签**精确相等**（不区分大小写），别名 `tag:react`               |
| `domain:github.com` | 来源域名   | 在来源 URL 的域名（hostname）中做子串匹配                              |
| `type:book`         | 来源类型   | 与来源类型**精确相等**（见下方枚举）                                   |
| `>=20260101`        | 起始日期   | 创建日期（本地）≥ 该日，别名 `after:` / `from:`                        |
| `<=20260630`        | 截止日期   | 创建日期（本地）≤ 该日，别名 `before:` / `to:`                         |
| `>20260101`         | 严格大于   | 创建日期（本地）> 该日                                                 |
| `<20260630`         | 严格小于   | 创建日期（本地）< 该日                                                 |
| `date:20260315`     | 指定某天   | 创建于该日当天                                                         |
| `in:2026`           | 整年范围   | 等价于 `>=20260101 && <=20261231`                                      |
| `~this-year`        | 本年范围   | 等价于 `in:<当前年份>`                                                 |

### 来源类型（type）枚举

`url`（网页）· `book`（书）· `conversation`（对话）· `experience`（经历）· `thought`（纯思考，无来源）

## 二、组合逻辑

支持布尔组合与括号分组：

- **且**：`AND`（或 `&&`），相邻子句省略运算符也默认为且。
- **或**：`OR`（或 `||`）。
- **非**：`NOT`（或 `!`），也支持前缀写法 `-`，如 `-type:book`。
- **优先级**：`NOT` > `AND` > `OR`。可用括号改变分组，例如 `(type:book OR type:thought) AND tag:react`。

## 三、引号与多词

值含空格时用引号包裹：

- `#"big idea"` —— 匹配标签 `big idea`
- `text:"machine learning"` —— 匹配正文中的 machine learning

## 四、日期与时间段

### 日期格式

日期统一使用 **YYYYMMDD**（8 位）。当前年份的日期可省略年份，只写 **MMDD**（4 位），例如 `>=0721` 意为 `>=20260721`。

### 比较运算符

- `>=20260101` —— 创建日期（本地）≥ 该日（别名 `after:` / `from:`）
- `<=20260630` —— 创建日期（本地）≤ 该日（别名 `before:` / `to:`）
- `>20260101` / `<20260630` —— 严格大于 / 小于

可组合：`>=20260101 && <=20260630` 表示 2026 年上半年。

### 时间段（in / ~）

`in:年份` 表示整年范围，等价于 `>=YYYY0101 && <=YYYY1231`。也支持 `~` 简写：

| 写法         | 含义            | 等价区间                   |
| ------------ | --------------- | -------------------------- |
| `~this-year` | 本年            | `in:<当前年份>`            |
| `~2020`      | 2020 全年       | `>=20200101 && <=20201231` |
| `~2026M3`    | 2026 年 3 月    | `>=20260301 && <=20260331` |
| `~M2`        | 本年 2 月       | 当前年份 2 月              |
| `~2026Q3`    | 2026 年第三季度 | `>=20261001 && <=20261231` |
| `~Q1`        | 本年第一季度    | 当前年份 1 季度            |

### 相对关键词

日期字段（`after` / `before` / `date` / `>=` 等）还支持相对关键词，按本地时间计算：

| 关键词       | 含义             |
| ------------ | ---------------- |
| `today`      | 今天             |
| `yesterday`  | 昨天             |
| `this-week`  | 本周（周一起算） |
| `this-month` | 本月             |
| `last-week`  | 上周             |
| `last-month` | 上月             |

示例：`>=this-month`（本月以来的微光）、`date:yesterday`（昨天的微光）。

> 注意：相对关键词只能作为**日期字段的值**使用（如 `date:today`），不能单独作为裸词——单独的 `today` 会被当作普通关键字去匹配文本。

## 五、示例

- `react` —— 查找任何包含「react」的微光
- `#family` —— 标签为 family 的微光
- `domain:github.com` —— 来源来自 github.com 的微光
- `type:book` —— 来源类型为「书」的微光
- `>=20260101 && <=20260630` —— 2026 年上半年的微光
- `#react OR #frontend` —— 标签为 react 或 frontend 的微光
- `-type:book` —— 排除来源为「书」的微光
- `#ml AND (type:book OR type:url)` —— 标签为 ml，且来源是书或网页
- `>=this-month` —— 本月以来的微光
- `in:2026` —— 2026 年全年的微光
- `~2026Q3` —— 2026 年第三季度的微光

## 六、语法错误

当查询无法解析（如未知字段 `foo:bar`、括号不匹配、字段缺值 `tag:`）时，不会崩溃，而是按原样回退为普通关键字匹配，保证搜索框始终可用。
